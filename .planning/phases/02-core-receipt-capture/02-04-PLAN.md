**1. Create `src/app/[locale]/(app)/receipt/[id]/verify/page.tsx`:**

Server component page within the `(app)` route group (inherits sidebar layout).

```typescript
import { headers } from 'next/headers'
import { notFound, redirect } from 'next/navigation'
import { setRequestLocale } from 'next-intl/server'
import { auth } from '@/lib/auth'
import { prisma } from '@/lib/db'
import { VerificationClient } from '@/components/receipt/verification-client'

type Props = {
  params: Promise<{ locale: string; id: string }>
}
```

Logic:
a. Extract `locale` and `id` from params, call `setRequestLocale(locale)`
b. Get session via `auth.api.getSession({ headers: await headers() })` — same pattern as dashboard
c. If no session: redirect to /auth
d. Fetch receipt: `prisma.capturedReceipt.findUnique({ where: { id, userId: session.user.id } })`
e. If not found: `notFound()`
f. If already verified (`verifiedAt` is not null): redirect to dashboard (receipt already processed)
g. Render `<VerificationClient receiptId={id} imageData={receipt.imageData} mimeType={receipt.mimeType} />`

Pass ONLY the image data needed for display — do NOT pass the full receipt object to the client component (minimize client bundle). The extraction action will read the image from DB server-side.

**2. Create `src/components/receipt/verification-client.tsx`:**

The main client orchestrator. This is the most complex component — manages the processing → verification → save flow.

Props:
```typescript
type VerificationClientProps = {
  receiptId: string
  imageData: string
  mimeType: string
}
```

State management:
```typescript
const [aiData, setAiData] = useState<ExtractionResult | null>(null)
const [aiLoading, setAiLoading] = useState(true)
const [aiTimedOut, setAiTimedOut] = useState(false)
const [showManualEntry, setShowManualEntry] = useState(false)
const [showAiReadyBanner, setShowAiReadyBanner] = useState(false)
const [saving, setSaving] = useState(false)
```

**Processing pipeline (useEffect on mount):**

```
On mount:
1. Set 10-second timeout
2. Call extractReceipt action with receiptId
3. If timeout fires before response:
   - Set aiTimedOut = true
   - Show: "This is taking longer than usual" + "Skip to manual entry" button
   - AI continues processing in background (action is still in-flight)
4. When action resolves (success):
   - Clear timeout
   - Set aiData = result
   - Set aiLoading = false
   - If showManualEntry is true (user already skipped):
     - Show AI-ready banner (NOT the stacked cards)
   - Else:
     - Show stacked cards with AI data
5. When action resolves (failure/error):
   - Clear timeout
   - Set aiLoading = false
   - Auto-switch to manual entry (showManualEntry = true)
   - Show toast with error message
```

Use `useAction` from next-safe-action/hooks for the extraction action. The `executeAsync` call is made in the useEffect.

**IMPORTANT per CONTEXT.md:** "AI continues processing in background after user skips to manual entry." This means do NOT abort the action when user clicks "skip to manual entry" — just update the UI state. The action's onSuccess callback handles the banner.

**UI rendering logic:**

```tsx
if (aiLoading && !showManualEntry) {
  // Phase 1: Processing screen
  return <ProcessingSkeleton imageData={imageData} mimeType={mimeType} timedOut={aiTimedOut} onSkip={() => setShowManualEntry(true)} />
}

// Phase 2: Verification screen (stacked cards)
return (
  <>
    {showAiReadyBanner && (
      <AIReadyBanner
        onMerge={() => { setShowAiReadyBanner(false); /* AI data already in state */ }}
        onDismiss={() => setShowAiReadyBanner(false)}
      />
    )}
    <ReceiptVerification
      imageData={imageData}
      mimeType={mimeType}
      aiData={aiData}
      aiLoading={aiLoading}
      onSave={handleSave}
      saving={saving}
    />
  </>
)
```

**Save handler:**
```typescript
const handleSave = async (data: VerifiedReceiptData) => {
  setSaving(true)
  try {
    await saveVerifiedReceiptAction({
      receiptId,
      ...data,
    })
    toast.success(t('receiptVerified'))
    router.push('/dashboard')
  } catch (error) {
    toast.error(t('saveFailed'))
    setSaving(false)
  }
}
```

Use `useRouter` from next/navigation for the redirect.
