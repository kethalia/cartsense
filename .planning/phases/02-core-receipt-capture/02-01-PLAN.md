```prisma
  // ── Extraction fields ──
  extractionStatus String    @default("pending")  // pending | processing | completed | failed
```

**3. Create `src/types/receipt.ts`** with ALL shared types used across the phase:

```typescript
/** Status of AI extraction for a receipt */
export type ExtractionStatus =
  | "pending"
  | "processing"
  | "completed"
  | "failed";

/** Payment type options */
export type PaymentType = "cash" | "card" | "other";

/** Result from AI extraction — returned by extractReceipt action */
export type ExtractionResult = {
  vendorName: string | null;
  totalAmount: number | null;
  receiptDate: string | null; // ISO date string (YYYY-MM-DD)
  taxAmount: number | null;
  paymentType: PaymentType | null;
  confidence: number; // 0-1 overall confidence
};

/** Field names available for source selection in verification UI */
export type ReceiptFieldName =
  | "vendorName"
  | "totalAmount"
  | "receiptDate"
  | "taxAmount"
  | "paymentType";

/** Source of field data — used in merge logic */
export type FieldSource = "manual" | "ai";

/** Map of each field to its selected source */
export type FieldSources = Record<ReceiptFieldName, FieldSource>;

/** Manual entry form data — string types for form inputs */
export type ManualEntryData = {
  vendorName: string;
  totalAmount: string; // string for input field, parsed to number on save
  receiptDate: string; // YYYY-MM-DD format
  taxAmount: string;
  paymentType: PaymentType | "";
};

/** Verified receipt data ready to persist */
export type ReceiptData = {
  vendorName: string;
  totalAmount: number;
  receiptDate: string | null; // ISO date string, null if not provided
  taxAmount: number | null;
  paymentType: PaymentType | null;
};

/** All 5 receipt field labels — for iteration in UI */
export const RECEIPT_FIELDS: readonly ReceiptFieldName[] = [
  "vendorName",
  "totalAmount",
  "receiptDate",
  "taxAmount",
  "paymentType",
] as const;
```

These types are the canonical contracts for the entire phase. Every other plan imports from this file.

**3. Create `src/actions/extract-receipt.ts`:**

Use `authActionClient` from `@/lib/safe-action` (same pattern as capture-receipt.ts). The action:

a. Accepts `{ receiptId: string }` via Zod schema
b. Reads the receipt from DB (verify it belongs to the authenticated user via `userId` from ctx)
c. Updates extractionStatus to "processing"
d. Constructs a Claude Vision API call:

- Model: `claude-sonnet-4-20250514` (cost-effective for structured extraction)
- System prompt: "You are a receipt data extractor. Analyze the receipt image and extract structured data. Return ONLY valid JSON, no other text."
- User message with the receipt image as base64 (`type: "image"`, `source.type: "base64"`, `source.media_type` from receipt mimeType, `source.data` from receipt imageData)
- User text: "Extract these fields from the receipt image:\n- vendorName: Store/vendor name\n- totalAmount: Total amount as a number (in RON)\n- receiptDate: Date in YYYY-MM-DD format\n- taxAmount: Tax/VAT amount as a number (in RON)\n- paymentType: 'cash', 'card', or 'other'\n\nReturn JSON with these exact field names. Use null for any field you cannot determine."
- max_tokens: 1024
  e. Parse Claude's JSON response. Wrap in try/catch — if JSON parsing fails, set extractionStatus to "failed" and return null.
  f. Map the parsed response to `ExtractionResult` type (import from `@/types/receipt`)
  g. Update the receipt record: set extractionStatus to "completed", populate vendorName/totalAmount/receiptDate/taxAmount/paymentType/confidence/rawExtraction
  h. Return the `ExtractionResult`

**Error handling:**

- If receipt not found or doesn't belong to user: throw error "Receipt not found"
- If Anthropic API fails: set extractionStatus to "failed", return `{ status: 'failed', error: string }`
- If JSON parsing fails: set extractionStatus to "failed", return `{ status: 'failed', error: 'Failed to parse AI response' }`
- If ANTHROPIC_API_KEY is not set: throw "AI extraction not configured"

**Important:** Initialize the Anthropic client at module level:

```typescript
const anthropic = new Anthropic(); // reads ANTHROPIC_API_KEY from env automatically
```

For the confidence score: if Claude returns a confidence field, use it. Otherwise, estimate based on how many fields were successfully extracted (e.g., 5/5 = 1.0, 3/5 = 0.6).

**4. Create `src/actions/save-verified-receipt.ts`:**

Use `authActionClient`. The action:

a. Accepts Zod schema:

```typescript
z.object({
  receiptId: z.string().min(1),
  vendorName: z.string().min(1, "Vendor name is required"),
  totalAmount: z.number().positive("Amount must be positive"),
  receiptDate: z.string().nullable(),
  taxAmount: z.number().min(0).nullable(),
  paymentType: z.enum(["cash", "card", "other"]).nullable(),
});
```

b. Verify receipt exists and belongs to user (same pattern as extract)
c. Update receipt record: set vendorName, totalAmount, receiptDate (parse to DateTime if provided), taxAmount, paymentType, verifiedAt to `new Date()`
d. Return `{ id: receipt.id, verifiedAt: receipt.verifiedAt }`

**Important:** For `totalAmount` and `taxAmount`, convert number to Prisma Decimal. Use `new Prisma.Decimal(value)` or just pass the number directly — Prisma handles the conversion.
