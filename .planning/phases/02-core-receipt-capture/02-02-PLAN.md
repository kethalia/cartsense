**1. Create `src/components/capture/fab-menu.tsx`:**

A client component that replaces the plain FabButton on the dashboard. Behavior:

- Default state: single circular FAB (Plus icon), same styling as existing FabButton (fixed bottom-6 right-6, h-14 w-14, rounded-full, shadow-lg)
- On tap: FAB icon rotates 45° to become an X (close icon), two option buttons animate upward from behind the FAB:
  - "Take Photo" button (Camera icon from lucide-react) — appears first, positioned above FAB
  - "Upload Image" button (Upload icon from lucide-react) — appears second, above the camera button
- Each option button: h-12 w-12 rounded-full, slightly smaller than main FAB, with a tooltip/label
- On option select: menu closes, callback fires
- On backdrop click or X tap: menu closes
- Use Tailwind transitions (scale, opacity, translate) for the animation — do NOT use motion/react here (save that for stacked cards). Use CSS transitions with `transition-all duration-200` and conditional classes.

Props:
```typescript
type FabMenuProps = {
  onTakePhoto: () => void
  onUploadImage: () => void
}
```

Include a semi-transparent backdrop (bg-black/20) when menu is open that closes on click.

Use `useTranslations('Camera')` for labels. The labels should be small text next to or below each option button.

**2. Create `src/components/capture/file-upload.tsx`:**

A client component with the same forwardRef + useImperativeHandle pattern as CameraCapture:

```typescript
export type FileUploadHandle = { trigger: () => void }
type FileUploadProps = {
  onUpload: (file: File, previewUrl: string) => void
  onError: (message: string) => void
}
```

- Hidden `<input type="file" accept="image/jpeg,image/png">` — NO `capture` attribute (opens file picker, not camera)
- On file selection, validate:
  a. **File type:** Must be image/jpeg or image/png. Reject with "Only JPG and PNG images are supported"
  b. **File size:** Must be ≤ 10MB. Reject with "Image too large. Maximum size is 10MB"
  c. **Resolution:** Load image into an HTMLImageElement, check naturalWidth × naturalHeight ≥ 300×300. Reject with "Image too small. Minimum size is 300×300 pixels"
- Resolution check is async (load image, read dimensions). Use a Promise wrapper.
- On validation pass: call `onUpload(file, previewUrl)`
- On validation fail: call `onError(message)`

onSuccess: ({ data }) => {
