Initialize Prisma:
```bash
npx prisma init --datasource-provider postgresql
```

Create the Prisma schema at `prisma/schema.prisma` with these models:

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        String   @id @default(cuid())
  clerkId   String   @unique
  email     String   @unique
  name      String?
  avatarUrl String?
  locale    String   @default("en")  // "en" or "ro"
  theme     String   @default("system")  // "light", "dark", or "system"
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  capturedReceipts CapturedReceipt[]
}

model CapturedReceipt {
  id         String   @id @default(cuid())
  userId     String
  imageData  String   @db.Text  // Base64 encoded image (Phase 1 temporary storage)
  mimeType   String
  fileSize   Int?     // Bytes
  capturedAt DateTime @default(now())
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId])
  @@index([capturedAt])
}
```

Key design decisions:
- `clerkId` on User links to Clerk's external user ID (not stored in our DB until first sync)
- `locale` and `theme` preferences stored per-user to persist across devices
- `imageData` as `@db.Text` stores base64 images — this is Phase 1 only; Phase 2 will migrate to cloud storage (Cloudinary)
- `CapturedReceipt` is intentionally simple — Phase 2 adds extracted data fields

Create `src/lib/db.ts` with the singleton PrismaClient pattern:
```typescript
import { PrismaClient } from '@prisma/client'

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

export const prisma = globalForPrisma.prisma ?? new PrismaClient()

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
```

Run Prisma generate (NOT push — that needs a running database):
```bash
npx prisma generate
```
