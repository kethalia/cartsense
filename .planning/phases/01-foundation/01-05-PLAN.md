**2. `src/components/capture/camera-capture.tsx`** — Camera capture component:
- 'use client' component
- Renders a hidden `<input type="file" accept="image/*" capture="environment" />` element
- Exposes a ref or method that the FAB can call to trigger the input click
- Per RESEARCH.md: Use `<input>` approach (NOT MediaDevices API) because:
  - Opens native camera directly on mobile
  - No permission prompts needed
  - Works on both iOS Safari and Android Chrome
  - Desktop falls back to file picker (appropriate for desktop upload)
- On file selected: reads the file, creates a preview URL, passes to PhotoPreview
- Implementation pattern:
  ```typescript
  const inputRef = useRef<HTMLInputElement>(null)
  const triggerCapture = () => inputRef.current?.click()
  const handleChange = (e: ChangeEvent<HTMLInputElement>) => {
    const file = e.target.files?.[0]
    if (file) {
      const previewUrl = URL.createObjectURL(file)
      setPreviewData({ file, previewUrl })
    }
  }
  ```

**3. `src/components/capture/photo-preview.tsx`** — Preview screen after capture:
- 'use client' component
- Full-screen overlay (or modal) showing the captured photo
- Two buttons only (per user decision):
  - "Use this" / "Folosește" — confirms and saves
  - "Retake" / "Refă" — dismisses preview, triggers camera again
- On "Use this":
  1. Convert file to base64: `const reader = new FileReader(); reader.readAsDataURL(file)`
  2. POST to `/api/receipts/capture` with body: `{ imageData: base64String, mimeType: file.type, fileSize: file.size }`
  3. Show loading state on button during upload
  4. On success: show sonner toast "Receipt saved! Processing coming soon." (localized)
  5. Close preview, refresh dashboard data (use `router.refresh()` from next/navigation)
- On "Retake": clear preview state, re-trigger camera input
- Image displayed at full width with aspect-ratio preserved
- Clean UI: image takes most of the screen, buttons at bottom

**4. `src/app/api/receipts/capture/route.ts`** — API endpoint to save captured receipt:
- POST handler:
  1. Get auth from `auth()` — reject with 401 if not authenticated
  2. Parse JSON body: `{ imageData: string, mimeType: string, fileSize?: number }`
  3. Validate: imageData exists, mimeType is image/* type, fileSize < 10MB
  4. Look up or create User record: Find user by clerkId, if not found create one (first API call creates the DB user record — sync Clerk user to local DB)
  5. Create CapturedReceipt record: `prisma.capturedReceipt.create({ data: { userId, imageData, mimeType, fileSize } })`
  6. Return 201 with `{ id, capturedAt }`
- Error handling: Return appropriate error responses with messages
