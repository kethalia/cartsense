Create the passwordless authentication components using Clerk's custom flow hooks.

**1. `src/components/auth/email-otp-form.tsx`** — Unified sign-in/sign-up with email OTP:
- This is a 'use client' component
- Uses `useSignIn()` from `@clerk/nextjs` for the primary flow
- Uses `useSignUp()` as fallback when the email doesn't have an existing account
- Two-step form:
  - Step 1: Email input field + "Continue" button
  - Step 2: OTP code input (use shadcn InputOTP component with 6 digits) + "Verify" button + "Resend code" link
- Flow logic:
  1. User enters email → call `signIn.create({ identifier: email })`
  2. If user exists → `signIn.prepareFirstFactor({ strategy: 'email_code', emailAddressId })` → show OTP input
  3. If user doesn't exist (Clerk returns specific error) → fall through to `signUp.create({ emailAddress: email })` → `signUp.prepareEmailAddressVerification({ strategy: 'email_code' })` → show OTP input
  4. User enters code → verify with appropriate handler (signIn or signUp)
  5. On success: `setActive({ session })` and redirect to home
- All text uses `useTranslations('Auth')` for i18n support. Import from 'next-intl'.
- Show loading states on buttons during API calls (use `isLoading` state)
- Show error messages inline below the relevant input (e.g., "Invalid code, please try again")
- Style: Clean, spacious form with generous padding. Use shadcn Button, Input, and InputOTP components.

**2. `src/components/auth/google-auth-button.tsx`** — Google OAuth one-click:
- 'use client' component
- Uses `useSignIn()` from `@clerk/nextjs`
- Single button: Google icon + "Continue with Google" text (localized)
- On click: `signIn.authenticateWithRedirect({ strategy: 'oauth_google', redirectUrl: '/sso-callback', redirectUrlComplete: '/' })`
- Style: Full-width outline button with Google logo icon
- Loading state while redirect is processing

Both components should handle errors gracefully — show toast notifications for unexpected errors, inline messages for validation errors.
