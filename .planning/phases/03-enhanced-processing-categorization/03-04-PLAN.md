**1. Create `src/lib/actions/search-receipts.ts`:**

Server action for searching receipts:

```typescript
const searchReceiptsSchema = z.object({
  query: z.string().max(200).optional(),
  categoryId: z.string().optional(),
  dateFrom: z.string().optional(),    // ISO date string
  dateTo: z.string().optional(),      // ISO date string
  amountMin: z.number().min(0).optional(),
  amountMax: z.number().optional(),
});
```

Implementation:
- Build Prisma `where` clause dynamically based on provided filters
- **Text search** (`query`): Use Prisma `OR` with `contains` (case-insensitive via `mode: 'insensitive'`):
  - `vendorName contains query`
  - `items: { some: { name: { contains: query, mode: 'insensitive' } } }` — searches product names
  - If query looks like a number (parseFloat succeeds): also match `totalAmount` equals
- **Category filter**: `categoryId: { equals: categoryId }`
- **Date range**: `receiptDate: { gte: dateFrom, lte: dateTo }` (use capturedAt as fallback if receiptDate is null)
- **Amount range**: `totalAmount: { gte: amountMin, lte: amountMax }`
- Order by: `receiptDate DESC` (or `capturedAt DESC`)
- Select same fields as the receipt list query (including category relation and imageData for thumbnails)
- Return matching receipts + a `matchContext` map: `{ [receiptId]: { field: string, matchedText: string }[] }` indicating which fields matched the text query

For `matchContext`: After the Prisma query, if `query` is provided, check each result:
- If vendorName contains query → add to matchContext
- If any item name contains query → add item name to matchContext
This enables highlighting on the client.

**3. Create `src/components/dashboard/receipt-search.tsx`:**

Search bar with integrated filter controls:

- **Search input**: Text input with search icon (Lucide `Search`), placeholder "Search receipts..."
  - Debounced input (300ms) — use a simple `useRef`/`setTimeout` debounce (no external library)
  - On change: call `searchReceipts` server action with current filter state
- **Filter buttons below search bar**: Tap to open filter popover/sheet
  - "Category" button → opens category picker (reuse CategoryPicker from Plan 03, but in filter mode — selecting sets the filter, doesn't update a receipt)
  - "Date" button → opens date range picker (two date inputs: from/to, with quick presets: "Today", "This week", "This month", "Last 30 days")
  - "Amount" button → opens amount range inputs (min/max RON values)
- Renders `FilterChips` below when any filters are active
- State: Use `useSearchParams` from next/navigation to persist search state in URL query params — enables back button and shareable URLs
- Passes search results to parent via callback: `onResults: (receipts, matchContext) => void`

The page should work both as a server component (initial load with search params) and handle client-side search via the ReceiptSearch component.

Approach: Use URL search params as the source of truth. ReceiptSearch updates URL params, which triggers a Next.js server component re-render with the new search results. This gives us:
- Server-side search (SEO + performance)
- Shareable/bookmarkable URLs
- Back button works

Use `useRouter().replace()` with updated search params (no page navigation, just URL update).
