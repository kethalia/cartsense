**1. Add Category model to prisma/schema.prisma:**

```prisma
model Category {
  id        String   @id @default(cuid())
  name      String                          // Display name (English)
  nameRo    String?                         // Romanian display name
  slug      String   @unique                // URL-safe identifier
  type      String                          // "receipt" | "product"
  color     String   @default("#6B7280")    // Hex color for chip
  icon      String?                         // Lucide icon name
  isCustom  Boolean  @default(false)        // User-created?
  userId    String?                         // null = system category
  sortOrder Int      @default(0)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user     User?             @relation(fields: [userId], references: [id], onDelete: Cascade)
  receipts CapturedReceipt[] @relation("ReceiptCategory")
  items    ReceiptItem[]     @relation("ItemCategory")

  @@unique([slug, userId])
  @@index([type])
  @@index([userId])
}
```

**2. Add ReceiptItem model** (persists line items currently stored in rawExtraction JSON):

```prisma
model ReceiptItem {
  id         String   @id @default(cuid())
  receiptId  String
  name       String
  quantity   Decimal  @db.Decimal(10, 3)   // Supports KG quantities like 0.456
  unitPrice  Decimal  @db.Decimal(10, 2)
  totalPrice Decimal  @db.Decimal(10, 2)
  vatRate    Decimal? @db.Decimal(4, 2)    // 19.00, 9.00, 5.00, or null (exempt)
  vatAmount  Decimal? @db.Decimal(10, 2)
  categoryId String?                       // Product-level category
  sortOrder  Int      @default(0)
  createdAt  DateTime @default(now())

  receipt  CapturedReceipt @relation(fields: [receiptId], references: [id], onDelete: Cascade)
  category Category?       @relation("ItemCategory", fields: [categoryId], references: [id], onDelete: SetNull)

  @@index([receiptId])
  @@index([categoryId])
}
```

**2. Create `src/lib/actions/category.ts`** with server actions using `authActionClient`:

- **`getCategories(type?: "receipt" | "product")`**: Returns all system categories + user's custom categories for the given type. Order by sortOrder, then name. No input schema needed (optional query filter).

- **`createCategory(input: CreateCategoryInput)`**: Creates a custom category for the current user. Sets `isCustom: true`, generates slug from name (lowercase, hyphenated, deduplicate with suffix if needed). Validates name length 1-50 chars. Returns the created category.

- **`updateCategory(input: { id: string } & UpdateCategoryInput)`**: Updates a category. Only allows updating categories where `isCustom: true AND userId = currentUser`. Throws if attempting to modify system categories. Returns updated category.

- **`deleteCategory(input: { id: string })`**: Deletes a custom category. Only allows deleting where `isCustom: true AND userId = currentUser`. Before deleting, set any receipts/items using this category to `categoryId: null`. Returns success boolean.

All actions use Zod input schemas from `src/schemas/category.ts`. Follow the existing `authActionClient` pattern from `src/lib/actions/capture-receipt.ts`.
